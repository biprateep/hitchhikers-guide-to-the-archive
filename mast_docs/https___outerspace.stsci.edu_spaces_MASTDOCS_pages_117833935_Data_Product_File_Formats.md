# File Format Summary
Most of the JWST science data files are images or tables in [FITS format](https://fits.gsfc.nasa.gov/fits_standard.html) (FITS 4.0 Standard), often with multiple extensions (MEF), while others are in some form of structured ASCII. Table 1 contains a short summary of the data product types that may be included with each data set along with the semantic content of the various data products, including some that are produced outside the science calibration pipeline.
Product type | Format | Description  
---|---|---  
[Science data](https://jwst-pipeline.readthedocs.io/en/stable/jwst/data_products/science_products.html) |  [FITS](https://fits.gsfc.nasa.gov/fits_standard.html) MEF images | Science data products for imaging, spectroscopy, coronagraphy, and (early stages of) time-series observations. The image extensions contain multi-dimensional science pixel arrays, concomitant data quality flags, and variance arrays. See the article on [Science Products](https://jwst-pipeline.readthedocs.io/en/latest/jwst/data_products/science_products.html) for details of the organization of information within these files.  
[FITS](https://fits.gsfc.nasa.gov/fits_standard.html) binary table | Extracted spectra are stored as tables, and include fields for spectral coordinates, flux, and concomitant quality information. [AMI](https://jwst-docs.stsci.edu/jwst-near-infrared-imager-and-slitless-spectrograph/niriss-observing-modes/niriss-aperture-masking-interferometry) fringe parameters are also stored as binary tables of coefficients.  
[ECSV](http://docs.astropy.org/en/stable/api/astropy.io.ascii.Ecsv.html) | Source catalogs, as well as time-series observation ([TSO](https://jwst-docs.stsci.edu/methods-and-roadmaps/jwst-time-series-observations)) light curves are stored in an ASCII _enhanced character-separated variable_ format.  
[ASDF](https://asdf-standard.readthedocs.io/en/latest/) | Metadata for images or spectra are stored in ASDF format, but are packaged as BYTE arrays in a FITS extension of type BINTABLE.  
[Guide-star data](https://jwst-pipeline.readthedocs.io/en/latest/jwst/guider_cds/description.html#id-mode) |  [FITS](https://fits.gsfc.nasa.gov/fits_standard.html) MEF images | Data products from associated guide-star activities: identification, acquisition, tracking and guiding. The image extensions are analogous to those for science data.  
[Associations](https://jwst-pipeline.readthedocs.io/en/stable/jwst/associations/index.html) | [ECSV](http://docs.astropy.org/en/stable/api/astropy.io.ascii.Ecsv.html) | A collection of metadata for all exposures that match a combination of attributes, including instrument, program ID, and observation, from which associations are generated within a given observing program.  
[JSON](http://www.json.org/) | Enumeration of relationships between different levels of science data products, and related calibration reference products, that were used during the course of calibration pipeline processing.  
[WSS](https://jwst-pipeline.readthedocs.io/en/stable/jwst/data_products/nonscience_products.html#wfs-c-combined-image-wfscmb) | [FITS](https://fits.gsfc.nasa.gov/fits_standard.html) | Wavefront-sensing Optical Path Difference (OPD) images that describe the state of the mirror segment alignments for a particular epoch. The OPD files can be used with a tool such as [WebbPsf](http://webbpsf.readthedocs.io/en/stable/) to determine the estimated PSFs for any given instrument aperture.  
Engineering data | CSV | Engineering data are provided in the form of a time series of values for a given telemetry point, in CSV format, with one series per file.  
[CAL reference files](https://jwst-pipeline.readthedocs.io/en/latest/jwst/references_general/index.html) |  [ASDF](https://asdf-standard.readthedocs.io/en/latest/), [ECSV](http://docs.astropy.org/en/stable/api/astropy.io.ascii.Ecsv.html), [FITS](https://fits.gsfc.nasa.gov/fits_standard.html), [JSON](http://www.json.org/) | Static reference files used in calibration processing come in various formats.  
Previews | JPEG or PNG | Preview and thumbnail images for presenting data products in the Portal.

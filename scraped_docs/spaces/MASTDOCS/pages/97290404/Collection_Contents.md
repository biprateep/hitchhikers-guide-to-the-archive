<!-- Metadata: {"Header 1": "Required Files"} -->
The contents of an HLSP collection are largely driven by the science outputs of the contributing team, but must include the following:
1. One or more**science files**. If the collection includes observational data, the files should be one of the types described in [File Types, Formats, Content](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/97290405/File+Types+Formats+and+Content) and include metadata within the files as described in [Required Metadata](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/97290409/Required+Metadata).
2. A**README**[example](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/97290408/Example+HLSP+README)) that must include:
1. contact information for the HLSP contributor
2. the contribution date
3. if applicable: the version identifier and a description of differences with the prior delivered version of the HLSP collection. We recommend a (lightly) structured format, but ASCII will do.
4. bibliographic information for one or more references to papers that describe the collection
3. A brief**project summary**

<!-- CHUNK 2 END -->

<!-- CHUNK 3 START -->
<!-- Metadata: {"Header 1": "Optional Content"} -->
An HLSP collectionmay also include:
* A**manifest**  
1. the location in the directory tree
2. the file name (see[naming convention](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/97290407/File+Naming+Convention))
3. The format should be structured to enable processing with scripts.
* Concomitant pixel data (variance arrays, bad pixel masks, exposure maps, etc.) where appropriate
* If the data originate from a NASA mission, provide a mapping between each HLSP data product and the mission source files from which they were derived. See the article on [Provenance Metadata](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/97290415/Provenance+Metadata).
* One or more graphics per science data product to be used as a preview; or a sky map for catalogs  
MAST staff will generate preview graphics for products such as images, spectra, and light curves using an automated process if not provided by the HLSP contributors. These auto-generated previews may be sub-optimal, however.

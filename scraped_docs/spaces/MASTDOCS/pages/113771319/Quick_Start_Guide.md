<!-- Metadata: {"Header 1": "Discovering JWST Data"} -->
The most effective approach to discovering JWST science data in MAST depends upon your objective and your involvement in JWST observing programs. If you are new to MAST or to using the Portal, see [MAST Primer for JWST](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/153686810/MAST+Primer+for+JWST).

<!-- CHUNK 3 START -->
<!-- Metadata: {"Header 1": "Discovering JWST Data", "Header 2": "Program PI or Co-I"} -->
If you are a Principal Investigator (PI) or Co-Investigator (Co-I) on one or more active JWST observing programs (i.e., approved programs that expect to obtain data in the current cycle), or are an Archive Researcher awaiting data from certain programs to become available, do one or more of the following:
* Consult the [Portal Guide](https://outerspace.stsci.edu/display/MASTDOCS/Portal+Guide) chapter [Program Subscriptions and Notifications](https://outerspace.stsci.edu/display/MASTDOCS/Program+Subscriptions+and+Notifications). Notifications will include a link to the associated observations, which when clicked will show these observations in the [Portal](https://mast.stsci.edu/portal/Mashup/Clients/Mast/Portal.html).
* Execute a [Portal Advanced Search](https://outerspace.stsci.edu/display/MASTDOCS/Advanced+Search) and tick the `JWST` box in the __Mission__ filter, and enter the program number in the __Proposal ID__ filter to search for data in your program.
* Review the [Using MAST APIs](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/113771434/Using+MAST+APIs) chapter in this manual to learn about scripted searches.

<!-- CHUNK 3 END -->

<!-- CHUNK 4 START -->
<!-- Metadata: {"Header 1": "Discovering JWST Data", "Header 2": "General User"} -->
If you have a scientific interest in JWST data but are not part of an investigator team, you might prefer these options:
* Execute a Portal  _[Advanced Search](https://outerspace.stsci.edu/display/MASTDOCS/Advanced+Search)_ and click the `JWST` box in the __Mission__ filter, or follow the [Using the MAST Portal](https://outerspace.stsci.edu/display/DraftMASTDOCS/.Using+the+MAST+Portal+v1.2) tutorial in this Manual for JWST-specific searches.
* Execute a Portal [basic search](https://outerspace.stsci.edu/display/MASTDOCS/Basic+Search) by target name or position to see if any JWST observations have been (or will be) obtained for sources of interest. The results will also include observations from all missions hosted at MAST.
* Review the [Using MAST APIs](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/113771434/Using+MAST+APIs) chapter in this manual to learn about scripted searches.
* Consult the list of [JWST Approved Programs](https://www.stsci.edu/jwst/science-execution/approved-programs) to see which science programs will obtain data in the current Cycle.  
Anonymous users will be able to search for all data, but will only be able to retrieve public data. See the next section.

<!-- Metadata: {"Header 1": "Available Science Data"} -->
Most science data obtained from JWST will be released to the astronomical community following an [**_exclusive access_** _**period**_](https://archive.stsci.edu/publishing/data-use#section-24a4fb60-93bc-4b19-803e-e8e7b04e8823) (EAP), during which the Principal Investigating Team enjoys exclusive scientific use. The duration of the exclusive access period depends upon the type of program. Some programs, whether by policy or choice, have a zero-length EAP.
Authorization for EAP Data
Principal Investigators are automatically given authorization to access data products within the EAP for their observing programs. PIs must authorize other members of their team if they need to access these data prior to the expiration of the EAP. See the article on [MAST User Accounts](https://outerspace.stsci.edu/display/MASTDOCS/MAST+User+Accounts) to learn how to grant access.

<!-- CHUNK 5 END -->

<!-- CHUNK 6 START -->
<!-- Metadata: {"Header 1": "Available Science Data", "Header 2": "Public Data"} -->
A large quantity of JWST data will be available for public use as soon as the observations are obtained and processed. These include the following categories of observing programs:
* **Early Release Observations:** Data from the ERO program, which demonstrate the capabilities of the JWST instruments, will be public after a brief embargo period.
* **Early Release Science:** Data from all 13 Cy-1 ERS programs have a zero-length EAP.
* **Cy-1 Guest Observer:** Data from 33 of the GO programs have zero-length EAP.
* **Cy-1 Guaranteed Time Observer:** Data from 16 of the GTO programs have a component with zero-length EAP.
* **Calibration:** All data obtained from JWST calibration programs have zero-length EAP; many of these include observations of astrophysical targets.
* **Commissioning** : All data obtained from JWST commissioning programs. Note that some commissioning data may not be suitable for science.
* All data for which the EAP has expired.

<!-- Metadata: {"Header 1": "Understanding JWST Data", "Header 3": "File Naming Scheme"} -->
The file name convention for JWST data products was designed to facilitate automated pipeline processing, and relates the files to details of the observing program that generated them. The names are not scientist-friendly. See the [Field Guide](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/113771321/Field+Guide+to+JWST+Data#FieldGuidetoJWSTData-FileNames) chapter for a summary, or the _**exposure**_ vs. those that relate to a _**source**_ or _**target**_.

<!-- Metadata: {"Header 1": "Understanding JWST Data", "Header 3": "Data Product Types"} -->
A data product type refers to the semantic content of the file. It is encoded in the product filename as a suffix, appearing just before the file extension (e.g., `*_cat.ecsv` indicates the file content is a source catalog). Which products you see for a given observation in MAST depends upon the instrument configuration (e.g., imaging, some form of spectroscopy, coronography), data taking mode (e.g., timeseries, dithers), and the level of processing. The calibration pipeline that produces JWST data products will be re-executed (and data products regenerated) after initial ingestion in response to events such as pipeline software updates or updates to calibration reference files. There are other reasons for a user to re-run the pipeline on their own system, e.g. to modify choices for backgrounds or spectrum extraction parameters. <!-- Metadata: {"Header 1": "Understanding JWST Data", "Header 2": "Selecting JWST Data Products"} -->
The full set of data products related to a single JWST observation may consist of hundreds of files. Standard MAST searches return **_Observations_** , which are collections of related files; this bundling de-clutters the interface and makes the selection of file types more manageable for users. See the chapter on [Data Product Linkages](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/113771324/Data+Product+Linkages) to see how files are rolled up under an Observation. One can use filters in the [Portal Download Basket](https://outerspace.stsci.edu/display/MASTDOCS/Download+Basket) to select the small number that are needed for actual science analysis. Using the "Minimum Recommended Products" filter is often helpful.

<!-- Metadata: {"Header 1": "Retrieving JWST Data"} -->
There are multiple options for retrieving data products from MAST. They include:
* Streaming data products from your Portal session
* Downloading a bash script to retrieve files asynchronously.
* Staging data at STScI for later retrieval
* Using third-party tools that support scripted data retrieval  
See the [Portal Guide](https://outerspace.stsci.edu/display/MASTDOCS/Portal+Guide) chapter on [Retrieving Data](https://outerspace.stsci.edu/display/MASTDOCS/Retrieving+Data) for details.

The most effective approach to discovering JWST science data in MAST depends upon your objective and your involvement in JWST observing programs. If you are new to MAST or to using the Portal, see [MAST Primer for JWST](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/153686810/MAST+Primer+for+JWST).

<!-- CHUNK 18 START -->
<!-- Metadata: {"Header 1": "Discovering JWST Data", "Header 2": "Program PI or Co-I"} -->
If you are a Principal Investigator (PI) or Co-Investigator (Co-I) on one or more active JWST observing programs (i.e., approved programs that expect to obtain data in the current cycle), or are an Archive Researcher awaiting data from certain programs to become available, do one or more of the following:
* Consult the [Portal Guide](https://outerspace.stsci.edu/display/MASTDOCS/Portal+Guide) chapter [Program Subscriptions and Notifications](https://outerspace.stsci.edu/display/MASTDOCS/Program+Subscriptions+and+Notifications). Notifications will include a link to the associated observations, which when clicked will show these observations in the [Portal](https://mast.stsci.edu/portal/Mashup/Clients/Mast/Portal.html).
* Execute a [Portal Advanced Search](https://outerspace.stsci.edu/display/MASTDOCS/Advanced+Search) and tick the `JWST` box in the __Mission__ filter, and enter the program number in the __Proposal ID__ filter to search for data in your program.
* Review the [Using MAST APIs](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/113771434/Using+MAST+APIs) chapter in this manual to learn about scripted searches.

<!-- Metadata: {"Header 1": "Discovering JWST Data", "Header 2": "General User"} -->
If you have a scientific interest in JWST data but are not part of an investigator team, you might prefer these options:
* Execute a Portal  _[Advanced Search](https://outerspace.stsci.edu/display/MASTDOCS/Advanced+Search)_ and click the `JWST` box in the __Mission__ filter, or follow the [Using the MAST Portal](https://outerspace.stsci.edu/display/DraftMASTDOCS/.Using+the+MAST+Portal+v1.2) tutorial in this Manual for JWST-specific searches.
* Execute a Portal [basic search](https://outerspace.stsci.edu/display/MASTDOCS/Basic+Search) by target name or position to see if any JWST observations have been (or will be) obtained for sources of interest. The results will also include observations from all missions hosted at MAST.
* Review the [Using MAST APIs](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/113771434/Using+MAST+APIs) chapter in this manual to learn about scripted searches.
* Consult the list of [JWST Approved Programs](https://www.stsci.edu/jwst/science-execution/approved-programs) to see which science programs will obtain data in the current Cycle.  
Anonymous users will be able to search for all data, but will only be able to retrieve public data. See the next section.

<!-- Metadata: {"Header 1": "Available Science Data"} -->
Most science data obtained from JWST will be released to the astronomical community following an [**_exclusive access_** _**period**_](https://archive.stsci.edu/publishing/data-use#section-24a4fb60-93bc-4b19-803e-e8e7b04e8823) (EAP), during which the Principal Investigating Team enjoys exclusive scientific use. The duration of the exclusive access period depends upon the type of program. Some programs, whether by policy or choice, have a zero-length EAP.
Authorization for EAP Data
Principal Investigators are automatically given authorization to access data products within the EAP for their observing programs. PIs must authorize other members of their team if they need to access these data prior to the expiration of the EAP. See the article on [MAST User Accounts](https://outerspace.stsci.edu/display/MASTDOCS/MAST+User+Accounts) to learn how to grant access.

<!-- CHUNK 20 END -->

<!-- CHUNK 21 START -->
<!-- Metadata: {"Header 1": "Available Science Data", "Header 2": "Public Data"} -->
A large quantity of JWST data will be available for public use as soon as the observations are obtained and processed. These include the following categories of observing programs:
* **Early Release Observations:** Data from the ERO program, which demonstrate the capabilities of the JWST instruments, will be public after a brief embargo period.
* **Early Release Science:** Data from all 13 Cy-1 ERS programs have a zero-length EAP.
* **Cy-1 Guest Observer:** Data from 33 of the GO programs have zero-length EAP.
* **Cy-1 Guaranteed Time Observer:** Data from 16 of the GTO programs have a component with zero-length EAP.
* **Calibration:** All data obtained from JWST calibration programs have zero-length EAP; many of these include observations of astrophysical targets.
* **Commissioning** : All data obtained from JWST commissioning programs. Note that some commissioning data may not be suitable for science.
* All data for which the EAP has expired.

The file name convention for JWST data products was designed to facilitate automated pipeline processing, and relates the files to details of the observing program that generated them. The names are not scientist-friendly. <!-- Metadata: {"Header 1": "Understanding JWST Data", "Header 3": "Data Product Types"} -->
A data product type refers to the semantic content of the file. It is encoded in the product filename as a suffix, appearing just before the file extension (e.g., `*_cat.ecsv` indicates the file content is a source catalog). Which products you see for a given observation in MAST depends upon the instrument configuration (e.g., imaging, some form of spectroscopy, coronography), data taking mode (e.g., timeseries, dithers), and the level of processing. <!-- Metadata: {"Header 1": "Understanding JWST Data", "Header 2": "Selecting JWST Data Products"} -->
The full set of data products related to a single JWST observation may consist of hundreds of files. Standard MAST searches return **_Observations_** , which are collections of related files; this bundling de-clutters the interface and makes the selection of file types more manageable for users. See the chapter on [Data Product Linkages](https://outerspace.stsci.edu/spaces/MASTDOCS/pages/113771324/Data+Product+Linkages) to see how files are rolled up under an Observation. One can use filters in the [Portal Download Basket](https://outerspace.stsci.edu/display/MASTDOCS/Download+Basket) to select the small number that are needed for actual science analysis. Using the "Minimum Recommended Products" filter is often helpful.

<!-- Metadata: {"Header 1": "Retrieving JWST Data"} -->
There are multiple options for retrieving data products from MAST. They include:
* Streaming data products from your Portal session
* Downloading a bash script to retrieve files asynchronously.
* Staging data at STScI for later retrieval
* Using third-party tools that support scripted data retrieval  
See the [Portal Guide](https://outerspace.stsci.edu/display/MASTDOCS/Portal+Guide) chapter on [Retrieving Data](https://outerspace.stsci.edu/display/MASTDOCS/Retrieving+Data) for details.
